## Модуль 3: Автоматический апсейл

### Ребёнок <8 лет:
- НЕ «немедленный апгрейд» — только **рекомендация проапгрейдиться**
- Конверсия: ~50/50 для групп, низкая для 1 взрослый + ребёнок

### Flexible Booking → триггер:
- Было: «тур через 5+ дней»
- **Должно быть: вплоть до 18:00 за день до поездки**

### Апгрейд premium shared при заполненности <50% за 48 часов:
- **Вопрос: зачем?** Теряем места на обычном shared
- Нужно обоснование или убрать

### Приоритет private для гостей из США/Австралии/Европы:
- «Европа» слишком широко (Румыния = Европа)
- **Ключевое**: shared зарабатывает БОЛЬШЕ чем private → предлагать private первым не всегда выгодно
- **Задача:** пересмотреть логику сегментации и апсейла

### Travel Insurance (ребрендинг Flexible Booking):
- ✓ Одобрено
- На OTA: риск нарушения правил (особенно Viator) → **изучить**

### Email fallback для +1/+61:
- Уточнение: если гость СНАЧАЛА написал нам → мы можем отвечать в WhatsApp
- Проблема только в том, что мы не можем ПЕРВЫМИ писать
- **Задача:** учесть этот кейс в логике, продумать подробнее
