### БЛОК 6.1 — HEADER: Статус флота

- **Данные:**  
  Агрегированные счётчики по всему флоту за текущий день:  
  - `X лодок активны` — лодки в статусе "В туре" прямо сейчас  
  - `Y на ТО` — лодки в статусе "Техобслуживание"  
  - `Z в резерве` — лодки готовы, но не назначены на тур  
  - Общее кол-во лодок в парке (знаменатель): `/17`  
  Пример: **"Флот сегодня: 11 активны / 3 на ТО / 3 в резерве — 17 лодок"**

- **Размер:** Полная ширина экрана. Высота ~56px. Фиксированная верхняя полоса.

- **Визуал:**  
  Тёмный фон (#1A1D2E или фирменный тёмно-синий).  
  Три pill-бейджа с цветовой кодировкой:  
  - 🟢 Активны — зелёный (`#22C55E`)  
  - 🟡 На ТО — жёлтый (`#EAB308`)  
  - 🔵 В резерве — синий (`#3B82F6`)  
  Каждый бейдж кликабелен → фильтрует грид ниже.  
  Справа в хедере: кнопка **[+ Новая лодка]** и аватар / имя David.

- **Обновление:** Real-time (WebSocket или SSE). Счётчики обновляются при смене статуса любой лодки.

- **Алерт-логика:**  
  Если `Y (на ТО) > 3` → фон бейджа "На ТО" становится красным, мигает 1 раз.  
  Если `X (активны) < 10` → бейдж "Активны" желтеет (ниже безопасного порога флота).

- **Действия:**  
  - Клик на бейдж "Активны" → фильтр грида по статусу "В туре"  
  - Клик на бейдж "На ТО" → фильтр по статусу "ТО"  
  - Клик на бейдж "В резерве" → фильтр по статусу "Готова"  
  - Клик на крестик фильтра → сброс, показать все

### БЛОК 6.2 — MAIN: Грид карточек лодок

- **Данные:**  
  Одна карточка = одна лодка. Данные из БД флота:
  - Название лодки (напр. "BLUUU-07 Nusa Spirit")
  - Фото (thumbnail 80×60px, из справочника лодок)
  - Текущий статус (В туре / Готова / ТО / Поломка)
  - Контекстные данные по статусу:
    - **В туре:** имя капитана, кол-во гостей на борту, расчётное время возврата
    - **ТО:** тип работ (напр. "Замена масла + фильтр"), ожидаемое время готовности
    - **Готова / Поломка:** последнее обслуживание, дата
  - Следующее плановое ТО: через X моточасов (из журнала моточасов)
  - Индикатор срочности ТО: зелёный (>50ч) / жёлтый (20–50ч) / красный (<20ч)

- **Размер:**  
  Основная область. Ширина: от левого края до начала правого сайдбара (~75% ширины экрана).  
  Грид: 4 колонки на Full HD, 3 на планшете, 2 на мобиле. Auto-fit.  
  Каждая карточка: ~220×180px.

- **Визуал:**  
  Карточка (Card component) с левой цветной полосой по статусу:  
  - В туре: зелёная полоса (#22C55E)  
  - Готова: синяя (#3B82F6)  
  - ТО: жёлтая (#EAB308)  
  - Поломка: красная (#EF4444) + пульсирующая анимация  

  Структура карточки (сверху вниз):
  ```
  [ФОТО 80×60] | BLUUU-07 Nusa Spirit
                | 🟢 В туре
  ──────────────────────────────────────
  Капитан: Made Suartha
  На борту: 12 гостей
  Возврат: 16:45
  ──────────────────────────────────────
  Следующее ТО: ⚠️ через 18 м/ч
  ──────────────────────────────────────
  [Поставить на ТО] [Сообщить о поломке]
  ```

  Для статуса "ТО":
  ```
  [ФОТО 80×60] | BLUUU-03 Coral Rider
                | 🟡 На ТО
  ──────────────────────────────────────
  Работы: Замена масла + фильтр
  Готовность: ~14:00
  Механик: Wayan Karsa
  ──────────────────────────────────────
  Следующее ТО: 🟢 через 95 м/ч
  ──────────────────────────────────────
  [Отметить готовой] [Сообщить о поломке]
  ```

  Для статуса "Поломка":
  ```
  [ФОТО 80×60] | BLUUU-11 Wave Dancer
                | 🔴 ПОЛОМКА
  ──────────────────────────────────────
  ⚠️ Требует внимания!
  Последнее обновление: 11:23
  ──────────────────────────────────────
  Следующее ТО: 🔴 ПРОСРОЧЕНО
  ──────────────────────────────────────
  [Взять в работу] [Обновить статус]
  ```

- **Обновление:** Real-time. При изменении статуса лодки карточка обновляется без перезагрузки страницы. Плавная анимация перехода цвета полосы.

- **Алерт-логика:**  
  - Поломка: карточка пульсирует красным, поднимается в начало грида  
  - ТО просрочено (моточасы = 0): иконка ⛔ на карточке  
  - Возврат из тура просрочен (>15 мин): счётчик времени краснеет  
  - Новая лодка без статуса: карточка с серым фоном "Не активирована"

- **Действия:**  
  - **[Поставить на ТО]** → модальное окно: выбрать тип ТО + механика + дату  
  - **[Отметить готовой]** → подтверждение одним кликом, статус → "Готова"  
  - **[Сообщить о поломке]** → модальное окно: описание + фото (drag & drop) + приоритет (критично/некритично)  
  - **[Взять в работу]** → назначить ответственного механика  
  - Клик на карточку → expand/детальная страница лодки (история ТО, инциденты, документы)

### БЛОК 6.3 — RIGHT SIDEBAR: Инциденты сегодня

- **Данные:**  
  Хронологическая лента всех инцидентов за текущий день:  
  - Время события (напр. 09:14)  
  - Название лодки  
  - Краткое описание (напр. "Утечка масла, двигатель правый борт")  
  - Статус решения: Новый / В работе / Решён  
  Счётчик вверху: "Сегодня: 3 инцидента (1 открыт)"

- **Размер:**  
  Правая колонка. Ширина: ~25% экрана. Высота: от header до нижней панели. Фиксированная, скроллируемая лента.

- **Визуал:**  
  Каждый инцидент — строка-карточка:
  ```
  09:14  BLUUU-07       [В работе 🟡]
         Утечка масла, правый двигатель
         → Механик: Ketut Wirawan
  ─────────────────────────────────────
  08:45  BLUUU-12       [Решён ✅]
         Якорная цепь заедает
  ─────────────────────────────────────
  07:30  BLUUU-05       [Новый 🔴]
         Не запускается мотор
         → Требует назначения механика
  ```
  Цвет левой полосы строки: красный (Новый) / жёлтый (В работе) / серый (Решён)  
  Кнопка **[+ Новый инцидент]** — закреплена сверху сайдбара, синяя, полная ширина сайдбара.

- **Обновление:** Real-time (push-уведомления от любого члена команды через BLUUU OS).

- **Алерт-логика:**  
  - Новый нерассмотренный инцидент → пульс-анимация красной полосы + звуковой сигнал (если включён)  
  - Инцидент без назначенного механика > 30 мин → строка выделяется красным

- **Действия:**  
  - **[+ Новый инцидент]** → форма: лодка (dropdown) + описание + фото + приоритет + ответственный  
  - Клик на инцидент → раскрывается детали + timeline + чат по инциденту  
  - Клик на статус инцидента → смена статуса (dropdown: Новый / В работе / Решён)

### БЛОК 6.4 — НИЖНЯЯ ПАНЕЛЬ: ТО-расписание на месяц

- **Данные:**  
  Плановое расписание технического обслуживания на текущий месяц.  
  Структура таблицы:  
  - Строки: лодки (17 лодок)  
  - Колонки: даты месяца (1–31)  
  - Ячейки: тип ТО + имя механика  
  Типы ТО: Плановое (синий) / Внеплановое (оранжевый) / Сезонное (фиолетовый)  
  Также показывает: суммарную загрузку механиков по дням (строка под таблицей)

- **Размер:**  
  Нижняя панель. Полная ширина (100%). Высота: ~250px. Горизонтальный скролл для дат.  
  Коллапсируемая: по умолчанию открыта, кнопка свернуть.

- **Визуал:**  
  ```
  Лодка          | 1    | 2    | 3    | 4    | ... | 30   | 31
  ─────────────────────────────────────────────────────────────
  BLUUU-01       |      | 🔵ПЛ |      |      |     |      |
  BLUUU-02       |      |      |      | 🟠ВН |     |      |
  BLUUU-03       | 🔵ПЛ |      |      |      |     | 🔵ПЛ |
  ...
  ─────────────────────────────────────────────────────────────
  Механик нагр.  | 1    | 2    | 0    | 1    |     | 1    | 0
  ```
  Легенда: 🔵 Плановое / 🟠 Внеплановое / 🟣 Сезонное  
  Текущая дата — выделена жёлтой вертикальной линией.  
  Кнопка **[+ Запланировать ТО]** — справа над таблицей.  
  Экспорт: **[Скачать PDF]** / **[Экспорт в Excel]**

- **Обновление:** Polling каждые 5 минут (редкие изменения).

- **Алерт-логика:**  
  - Ячейка с ТО сегодня + статус "Не начато" после 08:00 → красная рамка  
  - Лодка без плановых ТО >60 дней → строка подсвечивается оранжевым  
  - Превышение загрузки механика (>3 ТО/день) → ячейка нагрузки краснеет

- **Действия:**  
  - Клик на ячейку ТО → детали: тип работ, механик, материалы, статус  
  - **[+ Запланировать ТО]** → форма: лодка + дата + тип + механик + описание работ  
  - Drag & drop ТО на другую дату → перенос с подтверждением  
  - **[Скачать PDF]** → отчёт ТО за месяц для архива

# DASHBOARD 7 — GEEBRAN
## Head of Guides

> **Задача:** Расписание гидов + мониторинг качества через отзывы OTA.  
> **Боль:** Ручное планирование на каждый день, нет видимости рейтингов гидов в реальном времени.

### БЛОК 7.1 — HEADER: Покрытие гидами на завтра

- **Данные:**  
  - `X гидов нужно на завтра` — сумма всех туров завтра (каждый тур требует гида)  
  - `Y назначено` — гиды с подтверждённым назначением на завтра  
  - Разница `X - Y` = количество незакрытых позиций  
  - Дата: "Завтра, 3 апреля 2026"  
  - Статус: "✅ Все закрыты" / "⚠️ Не хватает N гидов"

- **Размер:** Полная ширина. Высота ~56px. Фиксированный хедер.

- **Визуал:**  
  Тёмный хедер. Левая часть: крупный текст с датой.  
  Центр: два бейджа-счётчика:  
  - "Нужно: **8**" (нейтральный синий)  
  - "Назначено: **6**" (зелёный если X=Y, красный если Y<X)  
  Если дефицит: красный баннер "⚠️ ВНИМАНИЕ: 2 гида не назначены!"  
  Справа: переключатель "Сегодня / Завтра / + 7 дней" (tabs)

- **Обновление:** Polling каждые 2 минуты (расписание меняется редко, но важно).

- **Алерт-логика:**  
  - `Y < X` → красный баннер в хедере  
  - `Y = X` → зелёный бейдж "Всё покрыто ✅"  
  - Незакрытый Premium-тур → отдельный красный баннер с пульсом

- **Действия:**  
  - Переключение вкладок "Сегодня / Завтра / +7 дней" → обновляет всю страницу

### БЛОК 7.2 — MAIN LEFT (60%): Расписание на неделю

- **Данные:**  
  Матрица: гиды (строки) × дни недели (колонки, пн–вс).  
  В каждой ячейке:
  - Название тура (напр. "Nusa Penida Full Day")
  - Тип тура (напр. "Premium" / "Standard" / "Snorkel")
  - Название лодки (напр. "BLUUU-07")
  - Время отправления (напр. "08:00")  
  Пустая ячейка = выходной гида.  
  Незаполненная ячейка, где нужен гид = ячейка с красным фоном "НУЖЕН ГИД".

  Список гидов (~12–15 человек): имя + фото-аватар 32px.  
  Строка итогов внизу: сколько туров в каждый день.

- **Размер:**  
  Левые 60% основной области (между хедером и нижней панелью).  
  Высота: ~400px. Вертикальный скролл если гидов много. Горизонтальный скролл для расширенного диапазона.

- **Визуал:**  

  Цветовая схема ячеек:
  - 🟢 Зелёный фон: гид назначен, тур подтверждён
  - ░░░ Серый: выходной (задан в графике)
  - 🔴 Красный фон + текст "НУЖЕН ГИД": слот без гида

  Строка с превышением загрузки (> 6 туров в день) → ячейка итога красная.

  - Красная ячейка "НУЖЕН ГИД" пульсирует если тур через <24ч  
  - Если Premium-тур без гида → ячейка с дополнительной иконкой ⭐ + красный

- **Действия:**  
  - Клик на заполненную ячейку → попап: детали назначения + кнопка "Заменить гида"

# SUMMARY: Дашборды 6–8

| Дашборд | Пользователь | Главный вопрос | Ключевое действие |
|---------|-------------|----------------|-------------------|
| **D6 David** | Head of Boats | Какие лодки работают прямо сейчас? | Поставить лодку на ТО / Сообщить о поломке |
| **D7 Geebran** | Head of Guides | Все туры завтра закрыты гидами? | Назначить гида / Посмотреть рейтинг |
| **D8 Widia** | Лаунж-менеджер | Все гости прибыли на посадку? | Отметить прибытие / Позвонить опоздавшему |

## Общие принципы UI для дизайнера

1. **Mobile-first для Widia** — она работает с телефоном в руках на пирсе. QR-скан должен быть доступен в 1 клик.
2. **Desktop-first для David и Geebran** — работают за компьютером в офисе. Грид лодок и таблица гидов требуют широкого экрана.
3. **Цветовая система едина** для всех дашбордов: 🟢 Зелёный = ОК / 🟡 Жёлтый = внимание / 🔴 Красный = проблема / 🔵 Синий = нейтральное действие.
4. **Real-time везде где важно** (статусы лодок, прибытие гостей). Polling там где изменения редки (рейтинги, ТО-расписание).
5. **Одно главное действие на дашборд:** David — статус лодки, Geebran — назначение гида, Widia — отметка прибытия. Все остальные действия вторичны.
6. **Алерты без шума:** красный = требует немедленного действия, жёлтый = надо проверить сегодня, нет цвета = всё ОК.
7. **Данные из единой БД:** все три дашборда смотрят на одни и те же данные броней, лодок, гидов — нет дублирования.
