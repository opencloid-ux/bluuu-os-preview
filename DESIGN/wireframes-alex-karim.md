> **Статус:** ТЗ для дизайнера
> **Версия:** 1.0
> **Дата:** 2026-04-02
> **Стек:** Next.js + Odoo backend + Respond.io (только гости) + n8n автоматизация

## ROW 1 — KPI-карточки (4 штуки, full-width)

> Четыре равные карточки (25% ширины каждая). Самая важная строка дашборда.

---

### [1.1] Выручка сегодня

- **Данные:**
  - Главное число: Rp 47.250.000 (IDR, крупный шрифт ~40px)
  - Подпись: "vs вчера: +12%" или "-8%"
  - Подпись: "vs план: 94% выполнено" (план на день — Rp 50.000.000)
  - Прогресс-бар под числом: заполнение = % выполнения дневного плана
- **Размер:** Quarter (25% ширины), высота ~140px
- **Визуал:**
  - Большое число + две строки с дельтой
  - Thin progress bar (высота 4px) внизу карточки
  - Иконка: монета / кошелёк
- **Обновление:** Каждые 5 минут (pull из Odoo)
- **Алерт-логика:**
  - Зелёный: выручка > 100% плана
  - Жёлтый: 70–99% плана (рабочий день до 16:00)
  - Красный: < 70% плана ИЛИ сильно ниже вчера (< -20%)
  - Серый нейтральный: до 10:00 утра
- **Действия:**
  - Клик → drill-down: разбивка по типам туров (снорклинг / манты / Килинкинг)
  - Hover → tooltip: "Снорклинг: Rp 28М | Манты: Rp 12М | Килинкинг: Rp 7.25М"

### [1.2] Туров сегодня

- **Данные:**
  - Главное число: 14 (количество туров запущено сегодня)
  - Статус GO/NOGO: большой badge "✅ GO" или "⛔ NO-GO"

## ROW 2 — График + Флот (2 колонки)

---

### [2.1] График выручки

- **Данные:**
  - Основной график: выручка по дням (IDR)
  - Переключатели вверху: "Неделя | Месяц | Год"
  - Режим "Неделя": 7 баров. Текущий день — голубой. Линия плана — пунктир серый.
  - Режим "Месяц": 30 точек, line chart. Факт (синий) и план (пунктир серый). Область под фактом полупрозрачная.
  - Режим "Год": 12 баров по месяцам. Текущий месяц незакрашен до конца (прогноз).
  - Легенда: Факт / План / Прогноз
  - Tooltip при hover: "Вт 31 марта: Rp 52.1М (+8% vs план)"
  - Под графиком: "Итого неделя: Rp 318М | vs прошлая неделя: +4.2%"
- **Размер:** 60% ширины, высота ~280px
- **Визуал:** Line + Bar chart (Recharts). Тёмная тема. Минималистичная сетка.
- **Обновление:** Каждые 15 минут
- **Алерт-логика:**
  - Если текущий день ниже плана > 20% — фон карточки слегка краснеет, заголовок "⚠️ Ниже плана"
  - Если рекорд дня/недели — badge "🏆 Рекорд"
- **Действия:**
  - Клик на бар/точку → drill-down по этому дню: разбивка по типам туров и источникам (Viator/Klook/Direct)
  - Кнопка "Экспорт CSV" в правом верхнем углу
  - Кнопка "Сравнить периоды" → overlay-график

---

### [2.2] Статус флота

- **Данные:**
  - Список всех лодок (17+), каждая — строка:
    - Название: Bluuu-01, Bluuu-02, ..., Bluuu-17
    - Тип: Speedboat / Traditional Jukung / Large Catamaran
    - Текущий статус (цветная точка + текст):
      - [ЗЕЛЁНЫЙ] "В туре" — лодка с гостями, GPS активен
      - [СИНИЙ] "Готова" — в порту, готова к следующему туру
      - [ЖЁЛТЫЙ] "ТО" — плановое техническое обслуживание
      - [КРАСНЫЙ] "Поломка" — требует ремонта, не использовать
      - [СЕРЫЙ] "Простой" — временно выведена из ротации
    - Капитан: имя
    - Если "В туре": маршрут + время возврата ("→ Манта Пойнт, вернётся 13:45")
  - Итоговая строка внизу: "Активных: 12 | В порту: 3 | ТО: 1 | Поломка: 1"
- **Размер:** 40% ширины, высота ~280px, скролл если лодок > 10
- **Визуал:** Список с цветными статус-точками. Компактные строки (~32px). Sticky footer с итогами.
- **Обновление:** Real-time (WebSocket / GPS polling каждые 2 мин)
- **Алерт-логика:**
  - Строка красная: статус "Поломка" или GPS потерян > 15 мин
  - Строка жёлтая: "ТО" или задержка возврата > 30 мин
  - Строка зелёная: всё в норме
  - Если > 2 лодок в статусе "Поломка" → красный заголовок блока + alert
- **Действия:**
  - Клик на лодку → боковая панель: история, лог маршрутов, контакт капитана
  - Клик "Поломка" → форма: описание + назначить механика
  - Кнопка "+ Добавить лодку" (только Admin)

  - [ЖЁЛТЫЙ] 08:30 — Погодный прогноз обновлён: волны 1.2м в 14:00–16:00. [Проверить маршруты]

  - Фильтры вверху: "Все | Критические | Предупреждения | Операционные | Бронирования"

- **Действия:**
  - Каждое событие: кнопка действия (контекстно: [Связаться] / [Просмотреть] / [Закрыть] / [Делегировать])
  - Кнопка "Отметить все прочитанными"
  - Кнопка "Экспорт лога за день"
  - Кнопка "Создать задачу" из любого события

  - Счётчик: "3 требуют действия сегодня"
- **Размер:** Full-width, высота ~220px, внутренний скролл
- **Визуал:** Timeline-лента с цветовой кодировкой. Каждый алерт: иконка типа + текст + кнопки конкретных действий. Возможность закрыть / отложить / делегировать.
- **Обновление:** Real-time WebSocket + периодический check (каждые 5 мин) из Odoo на новые несоответствия
- **Алерт-логика:**
  - Красные: всё что блокирует операции (овербукинг с гостями, сломанная лодка с туром, кассовый разрыв сегодня/завтра)
  - Жёлтые: нужно решить в течение дня (нет гида, просроченный invoice, технические проблемы)
  - Синие: возможности и входящие запросы
- **Действия:**
  - Каждый алерт: 1–3 контекстных кнопки действия (прямо в карточке)
  - Кнопка "Закрыть" / "Отложить на 2ч" / "Делегировать →"
  - Кнопка "Создать задачу" → алерт превращается в задачу с дедлайном
  - Все действия логируются в ленте событий

## Примечания по дизайну для обоих дашбордов

### Цветовая система

| Цвет | Hex | Когда использовать |
|------|-----|-------------------|
| Тёмно-синий фон | #0A1628 | Основной фон страницы |
| Синий акцент | #1E6BC4 | Кнопки, иконки, активные элементы |
| Голубой | #4AAED9 | Текущие/активные статусы |
| Зелёный | #22C55E | Норма, успех, GO |
| Жёлтый/Оранжевый | #EAB308 | Предупреждение, требует внимания |
| Красный | #EF4444 | Критично, требует немедленного действия |
| Серый | #6B7280 | Неактивные элементы, завершённые туры |
| Белый | #F8FAFC | Основной текст на тёмном фоне |

### Типографика

- Заголовки блоков: Inter SemiBold, 14px, uppercase, letter-spacing 0.05em
- Большие числа KPI: Inter Bold, 36–42px
- Подписи и метки: Inter Regular, 12–13px, #94A3B8
- Кнопки действий: Inter Medium, 12px

### Breakpoints и адаптивность

- Desktop 1440px+: полная раскладка как в схемах
- Laptop 1280px: ROW 3 сжимается (меньше padding)
- Tablet 1024px: ROW 3 → 2 колонки + scroll, таблица туров горизонтальный scroll
- Mobile: отдельный мобильный дашборд (только критические KPI + алерты)

### Интерактивность

- Все карточки: hover → лёгкая подсветка (box-shadow: 0 0 0 1px #1E6BC4)
- Клик на любой блок (если нет кнопки) → drill-down страница
- Таблица: row click → slide-in боковая панель с деталями
- Алерты: новые появляются сверху с animate-slide-in + fade

### Обновление данных

- WebSocket подключение для real-time данных (статусы лодок, алерты)
- React Query с polling интервалами для менее критичных данных
- Визуальный индикатор "Last updated: X min ago" в footer каждого блока
- При потере соединения: toast "Нет связи с сервером" + все данные становятся серыми
