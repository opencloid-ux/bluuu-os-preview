# BLUUU OS — Wireframes: GR Dashboards

## Дашборд 3: Дима (Guest Relations Director)

### HEADER
- **Данные:**
  - Количество активных чатов (число)
  - Количество активных эскалаций (красный бейдж с числом)
  - Количество pending задач
- **Размер:** top (full-width, ~60px по высоте)
- **Визуал:** Toolbar с цифрами и бейджами
- **Обновление:** real-time
- **Алерт-логика:**
  - Бейдж эскалаций становится ярко-красным при появлении новых/необработанных случаев
  - Бейдж задач пульсирует при срочных задачах
- **Действия:**
  - Клик по бейджу открывает соответствующую панель (эскалации/задачи)

---

### LEFT SIDEBAR: Очередь эскалаций от бота
- **Данные:**
  - Список гостей (имя/ID, краткая информация), которых бот не смог обработать
  - Причина эскалации (текст: цена/жалоба/VIP/другое)
  - Время ожидания (таймер с момента попадания в эскалацию)
- **Размер:** Left, 25% ширины, full height
- **Визуал:** Список карточек, отсортировано по времени ожидания (сверху дольше всего ждать)
- **Обновление:** real-time
- **Алерт-логика:**
  - Строка становится красной, если ожидание превышает SLA (напр. >10 минут)
  - Вибро/звук при эскалации VIP/жалобы
- **Действия:**
  - Кнопка "Взять в работу" для каждой эскалации → гость фиксируется за этим оператором, уходит из очереди

---

### MAIN CENTER: Активный чат с гостем
- **Данные:**
  - Переписка с гостем (история сообщений из Respond.io через API)
  - Контекст гостя (имя, тур, дата, оплата, особые требования)
  - Быстрые ответы (набор шаблонов сообщения с категорией)
  - Кнопки действий: "Подтвердить бронь", "Запросить оплату", "Эскалировать Кариму"
  - AI-подсказка: поле с предлагаемым AI черновиком ответа (и кнопка быстро вставить)
- **Размер:** Center, 50% ширины, full height
- **Визуал:**
  - Чат-блок (bubble/chat)
  - Контекст гостя — выведен над/history или в боковой панели внутри блока
  - Быстрые ответы/AI-подсказка — отдельный блок/панель справа в MAIN CENTRE
- **Обновление:** real-time
- **Алерт-логика:**
  - Если новое сообщение — highlight/всплытие
  - Если AI-подсказка обновлена — маленький анимационный индикатор
- **Действия:**
  - Клик"быстрый ответ" → вставка шаблона в поле ввода
  - Кнопки действий → всплывающее окно подтверждения
  - AI draft → вставить текст в чат/редактировать

---

### RIGHT PANEL: Карточка гостя
- **Данные:**
  - Имя, страна/город, язык общения
  - Тур: дата тура, тип тура, лодка, гид
  - Статус оплаты (оплачен/не оплачен/частично)
  - Специальные требования (чекбоксы/текст: беременная, дети, веган и т.д.)
  - История — был ли раньше, отзывы (кратко)
- **Размер:** Right, 25% ширины, full height
- **Визуал:** Вертикальная карточка с ключевой инфой, акценты/бейджи/иконки
- **Обновление:** По клику на чат/гостя — подгружается 1 раз, далее real-time обновление статуса оплаты
- **Алерт-логика:**
  - Красный индикатор, если отказ/проблема по оплате/особое требование
- **Действия:**
  - Клик по любому блоку открывает детальное модальное окно/историю

---

### НИЖНЯЯ ПАНЕЛЬ: Завтрашний тур чеклист
- **Данные:**
  - Таблица: все гости на завтра (имя, тур, номер лодки, гид, статус водителя, чек-ин заполнен [да/нет], оплата ОК [да/нет])
  - Красным выделено то, что не готово (статус "нет" для чекина/оплаты/водителя)
- **Размер:** Bottom, full-width, height ~200-300px
- **Визуал:** Таблица с фильтрацией/сортировкой, кнопка "Отправить всем напоминание"
- **Обновление:** polling каждые 10 минут
- **Алерт-логика:**
  - Если есть хоть одна строка в красном (что-то не готово) — панель outline красным/звуковой сигнал
- **Действия:**
  - Кнопка "Отправить всем напоминание" — массовая отправка напоминаний по всем неподтверждённым гостям

## Дашборд 4: GR Менеджер

### HEADER
- **Данные:**
  - Кол-во гостей сегодня (моих)
  - Кол-во открытых задач (по мне)
- **Размер:** Top, full-width, ~60px
- **Визуал:** Toolbar с цифрами/бейджами
- **Обновление:** real-time
- **Алерт-логика:**
  - Красный бейдж/анимация при увеличении проблем
- **Действия:**
  - Клик по цифре — фильтр на панели гостей/задач

---

### LEFT: Мои активные диалоги
- **Данные:**
  - Список чатов: имя гостя, последнее сообщение, время последнего обновления
  - Цветовая индикация: красный (>30 мин ожидания), жёлтый (>10 мин), зелёный (<10 мин)
  - Фильтры: Новые/Активные/Закрытые
- **Размер:** Left, 30% ширины, full height
- **Визуал:** Список/таблица чатов, фильтры tabs сверху
- **Обновление:** real-time
- **Алерт-логика:**
  - Цвет блока меняется по SLA
- **Действия:**
  - Клик → открыть чат в центре

---

### CENTER: Активный чат + карточка гостя
- **Данные:**
  - История чата
  - Карточка гостя (аналогично как у Димы, суженная)
- **Размер:** Center, 70% ширины, full height
- **Визуал:** Чат+карточка, 2 панели side-by-side
- **Обновление:** real-time
- **Алерт-логика:**
  - Новый чат/сообщение — подсветка
- **Действия:**
  - Кнопки шаблонных ответов, как у Димы

---

### НИЖНЯЯ ПАНЕЛЬ: Мой список задач на сегодня
- **Данные:**
  - Call-лист, список по задачам: кому позвонить, кому отправить подтверждение, кому нужен водитель завтра (имя + действие)
- **Размер:** Bottom, full-width, ~150px
- **Визуал:** Чек-лист/список
- **Обновление:** при изменении задач/выполнении — real-time
- **Алерт-логика:**
  - Строка выделяется жёлтым/красным — просрочено
- **Действия:**
  - Чекбокс "выполнено"

## Дашборд 5: Aline (OTA Manager)

### HEADER
- **Данные:**
  - "Синхронизация OTA: последний раз X минут назад"
  - Статус Channel Manager (active/down/error, бейдж)
- **Размер:** Top, full-width, ~60px
- **Визуал:** Строка статуса + бейдж
- **Обновление:** polling/real-time
- **Алерт-логика:**
  - Красный/жёлтый бейдж при ошибке
- **Действия:**
  - Клик по времени — открыть логи синхронизаций

---

### MAIN TABLE: Матрица доступности
- **Данные:**
  - Таблица: строки — даты (30 дней вперёд), столбцы — Shared Std, Shared Premium, Private, Bundled, Yacht
  - Ячейка: число мест, цвет (зелёный/жёлтый/красный), статус (открыто/закрыто)
- **Размер:** Main, full-width, 450-600px высота
- **Визуал:** Таблица-матрица, clickable ячейки
- **Обновление:** polling каждые 15 минут
- **Алерт-логика:**
  - Красная ячейка — мало мест / закрыто
- **Действия:**
  - Клик по ячейке — открыть/закрыть слот с подтверждением

---

### ROW 2 LEFT: Синхронизация по платформам
- **Данные:**
  - Viator: статус (иконка: ✅/⚠️)
  - Klook, GYG, Airbnb — аналогично
  - Кнопка "Принудительная синхронизация"
- **Размер:** Left, 50% ширины второй строки
- **Визуал:** Вертикальный список + кнопка
- **Обновление:** real-time
- **Алерт-логика:**
  - При ошибке — строка красная/иконка ⚠️, звук
- **Действия:**
  - Клик по кнопке — запуск синхронизации, появляется спиннер/статус

---

### ROW 2 RIGHT: Flash Sale панель
- **Данные:**
  - Форма: дата, % скидки, выбор платформ
  - Список активных акций (дата, платформа, скидка)
- **Размер:** Right, 50% ширины второй строки
- **Визуал:** Форма + список
- **Обновление:** real-time
- **Алерт-логика:**
  - Акция истекает/ошибка — подсветка
- **Действия:**
  - Кнопка "Запустить акцию" — появляется запись в списке
