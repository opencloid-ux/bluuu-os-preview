## 2. Топ повторяющихся ежедневных процессов

### Процесс 1: Утренний Check-in (Daily_Working_Guides + Guides_1)
**Время:** ~07:00–10:00 каждый день
**Кто:** Каждый гид вручную пишет в чат:
```
[Лодка] [Тип тура] [N pax] check in / complete / ready to go
```
Иногда прикладывают фото борта. MOD следит, что все лодки вышли.

### Процесс 2: Ежедневный анонс обедов (Amarta + Khamara + La Bianca)
**Время:** ~08:30–10:30 каждый день
**Кто:** Gee вручную составляет и отправляет 2-3 сообщения в разные чаты:
```
Halo kak, Berikut update lunch [Amarta/Khamara] today:
Shared Tour: [лодка _ N pax]
Private Tour: [лодка _ N pax]
```
Плюс dietary restrictions, bday cakes, buffet vs. set menu.

### Процесс 3: Вечерний Trip Report (Daily_Working_Guides)
**Время:** ~15:00–19:00 каждый день
**Кто:** Каждый гид вручную пишет полный отчёт по шаблону:
- Дата, лодка, кол-во пакс, гиды
- Полный маршрут с точными временами каждой точки
- Статус снаряжения (towels, gear complete)
- Особые события (no manta, rough sea, guest sick, early return)

### Процесс 4: Оплата ретрибуции Nusa Penida (Daytrip_tiket_online)
**Время:** Каждый день тура (утро + afternoon)
**Кто:** Gee → анонс групп → офицеры → фото тикетов → Kassy → оплата → Gee → подтверждение перевода. 5+ шагов вручную.

### Процесс 5: Погодный брифинг (Daily_Working_Guides)
**Время:** ~06:00–08:00 ежедневно
**Кто:** Gee отправляет сводку (swell meters, wind knots) с рекомендациями для конкретных точек. Гиды реагируют реал-тайм ("choppy", "balik kanan", "mau utara").

### Процесс 6: GoPro Footage Distribution (Guides_1)
**Время:** Каждый вечер / следующее утро
**Кто:** Komang Ryan Driver / Weda собирают footage вручную, создают Google Drive папки, публикуют ссылки для каждой пары гидов. Гости иногда жалуются на битые ссылки.

### Процесс 7: Инвойсинг обедов (La_Bianca + Khamara)
**Время:** Еженедельно
**Кто:** Ручная сверка: кто приехал → инвойс PDF → скрин перевода. Накапливаются задолженности за месяцы (в La Bianca была ситуация с неоплаченными счетами с ноября по январь).

## 3. Болевые точки и ручная работа (что можно автоматизировать)

### 🔴 Критично — потеря денег / репутации

**3.1 Несоответствие pax count при обедах**
- Постоянная проблема: в инвойсе одно число pax, в чате другое. Пример: La Bianca выставила за 5, фактически приехало 6 (+ фотограф).
- Причина: нет единой системы учёта. Данные из Whatsapp → ресторан → инвойс — везде разные.
- Решение: центральная база данных бронирований с автоматической передачей pax count в рестораны.

**3.2 Долги по оплате обедов**
- La Bianca ждала оплату с ноября по январь (2+ месяца). Khamara регулярно сталкивается с поздними отменами без компенсации.
- Причина: нет автоматического трекинга задолженностей, ручное сведение раз в неделю/месяц.

**3.3 Отмены в последний момент без уведомления ресторанов**
- Частый паттерн: группа отменяет обед уже на месте или на воде → ресторан готовил на N pax.
- Решение: автоматическое уведомление ресторанов при изменении статуса группы.

### 🟠 Высокий приоритет — операционные потери

**3.4 Ежедневный мануальный анонс обедов × 3 ресторана**
Gee каждый день составляет одни и те же сообщения в 3 разных чата (Amarta, Khamara, La Bianca). Информация одна и та же — лодка + pax + dietary. Занимает 30-60 минут + постоянные correction updates.
- Решение: одна форма в BLUUU OS → автоматическая рассылка всем ресторанам.

**3.5 Ретрибуция — полностью ручной процесс**
5 шагов вручную каждый день: анонс → фото тикетов → оплата онлайн → подтверждение → сверка. Технические сбои ATIX/BPD bank добавляют хаоса.
- Решение: интеграция с ATIX API или автоматическое формирование платёжного запроса по данным бронирования.

**3.6 Trip reports — ручной шаблон 25+ гидов каждый вечер**
Каждый гид пишет один и тот же structured report, но в свободной форме. Качество варьируется сильно: одни пишут точные времена каждой точки, другие — минимум. MOD тратит время на парсинг.
- Решение: форма в приложении с обязательными полями (маршрут, тайминги, инциденты, снаряжение).

**3.7 Check-in утром — нет верификации**
Гид написал "14 pax complete" — но никто не проверяет соответствие с бронированием. Часто: "only 12 pax, 2 not coming", "add 1 kid", "guest sick".
- Решение: цифровой чеклист при check-in с фото борта.

### 🟡 Средний приоритет

**3.8 GoPro footage — ручной сбор и раздача**
Каждый день Ryan/Weda вручную собирают footage, создают папки Google Drive, публикуют ссылки в чат. Гости жалуются на ошибки в папках (файлы с "._" в имени).
- Решение: автоматическая структура папок по дате/гиду + уведомление гостям.

**3.9 Dietary restrictions — теряются в чате**
Dietary info (vegan, nut allergy, gluten free) отправляется в текстовом сообщении. Ресторан может пропустить, особенно если пришёл поздний update.
- Решение: отдельное поле в системе бронирования → автоматически в анонс ресторана.

**3.10 Weather decision — нет формальной системы**
Решение "ехать / не ехать" принимается утром в чате на основе личного суждения Gee. Нет формальной системы пороговых значений (swell > X метров = отмена).

**3.11 Inbound cancellations — управление вручную**
"Guest sick, not coming" — гид сообщает в чат, MOD должен обновить: ресторан, ретрибуцию, бухгалтерию. Каждый канал — отдельное ручное сообщение.

## 4. Финансовые процессы

### 4.1 Оплата обедов в Amarta
- Механизм: скорее всего включено в пакет тура, ресторан считает по головам на месте
- Проблема: нет инвойсинга от Amarta в чате (только подтверждения), значит возможно прямые счета офф-чат
- Dietary extras (cakes, dry couch) — не упоминается отдельная оплата

### 4.2 Оплата обедов в Khamara
- Механизм: Gee отправляет список → ресторан готовит → оплата, судя по всему, постфактум или по договору
- Проблема: late cancellations (Sanjaya 1 skip lunch, LM group cancel due to rain) — неясно, берётся ли cancellation fee
- Потерянное полотенце разбиралось через чат Khamara (нашли / не нашли)

### 4.3 Инвойсинг La Bianca
- Механизм: Gee отправляет update → La Bianca PDF-инвойс → DTB еженедельный перевод → скрин в чат
- Формат инвойсов: USD-173, USD-174... (последовательная нумерация)
- Задержки: были инвойсы за ноябрь–январь без оплаты (выявлено только при составлении recap)
- Контроль: ручная Excel-таблица (Rekap Payment La Bianca), красный = не оплачено, жёлтый = оплачено
- Несоответствия pax: фиксировались как минимум дважды в первые же дни после создания группы

### 4.4 Ретрибуция Nusa Penida
- Тариф: взрослые (иностранцы), дети 1-9 лет — бесплатно, от 10 лет — полный тариф
- Механизм оплаты: trip.com или ATIX система → QR-код / скрин → в чат
- Банковский счёт провайдера: BRI 474501035104538 (I Gede Sunata Juna Hanggara)
- Частота: ежедневно, per-boat, конец дня
- Проблемы: last-minute pax changes, система ATIX/BPD падает, детские тикеты надо корректировать вручную

### 4.5 Компенсации гидам
- Упоминается в Guides_1: при массовых отменах — 50% cancellation fee гидам, при продолжении отмен — 25%
- Alex Boss публично объявлял в чате (не в системе)

## 5. Рекомендации для BLUUU OS Dashboard Geebran

### 5.1 🗓️ Daily Briefing Module
**Что:** Автоматическая подготовка утреннего анонса для ресторанов на основе данных бронирований.
- Input: система бронирований → список лодок + pax + dietary flags на сегодня
- Output: автоматически сформированные сообщения → отправка в Amarta, Khamara, La Bianca (через WhatsApp API или вручную 1 кнопкой)
- Gee тратит на это ~45 мин/день → сократить до 2-3 мин (review + send)

### 5.2 ✅ Digital Check-in / Trip Reporting
**Что:** Мобильная форма для гидов (в BLUUU OS app):
- Check-in: лодка, тип, pax count, фото борта → автоматически обновляет статус в системе
- Departure confirmation: все ли на борту
- Trip report вечером: обязательные поля маршрута с временами, снаряжение, инциденты
- Ценность: структурированные данные вместо free-text → аналитика, автоматические обновления смежных систем

### 5.3 💰 Lunch Tracking & Invoice Management
**Что:** Единый дашборд для отслеживания обедов и инвойсов:
- Ежедневный log: какие группы обедали, в каком ресторане, сколько pax (включая dietary)
- Reconciliation: система сравнивает pax в бронировании vs. фактических посетителей
- Invoice tracker: статус каждого инвойса (received / paid / overdue)
- Auto-alert: если инвойс не пришёл в течение 48 часов после визита

### 5.4 🎫 Retribusi Auto-Payment
**Что:** Автоматизация процесса ретрибуции:
- Утренний анонс провайдерам генерируется из бронирований (только darat-группы)
- Оплата: автоматический batch payment через ATIX API (или ready-to-paste payment summary)
- Трекинг: какие тикеты оплачены, какие pending, история по датам

### 5.5 📊 Guide Performance Dashboard
**Что:** Ежемесячный рейтинг гидов (сейчас Gee считает и постит вручную):
- Источники данных: Google Reviews, Tripadvisor, внутренние оценки
- Автоматический расчёт + публикация в конце месяца
- History: динамика по месяцам, выявление падений

### 5.6 🌊 Weather Decision Support
**Что:** Утренний weather briefing модуль:
- Интеграция с weather API (swell, wind, visibility)
- Пороговые значения: swell > 2m + wind > 15 knots = yellow alert, > 3m = red (recommend cancel)
- Автоматическое уведомление всем гидам + ресторанам о статусе тура

### 5.7 🔔 Cancellation Cascade
**Что:** При отмене тура или группы — одно действие обновляет все смежные каналы:
- Ресторан (lunch cancel)
- Ретрибуция (меньше pax)
- Гид (cancel notification + compensation trigger)
- Бухгалтерия (скорректированный invoice expected)

### 5.8 📹 GoPro Footage Tracker
**Что:** Автоматическая структура хранения footage:
- Папки создаются автоматически по дате + имени гида (из расписания)
- Уведомление гостям с ссылкой — автоматически через 24 часа после тура
- Ошибка с "._" файлами — правило фильтрации при загрузке

## 6. Ключевые цитаты и паттерны из чатов

### Паттерн: утренний check-in (ежедневно × 25+ гидов)
```
[07:36] Sandy Guide DTB: Shared trip Jayanadi 13 pax complete ✅
[07:41] Timpleng Dtb: My darling private boat 7pax check-in
[07:47] Micky: SH 4 Shering trip 14 pax complete 🙏
```
→ **Каждый день ~25 отдельных ручных сообщений в формате, который система должна собирать автоматически.**

### Паттерн: ежедневный обеденный анонс (ежедневно × 3 чата)
```
[08:41] Gee DTB: Selamat pagi Amarta team,
Berikut update lunch 01-04-2026:
Shared - RikiJ _ 13, SH4 _ 12
Private - SD _ 8
Thankyou
```
→ **Одна и та же информация отправляется в 3 места вручную каждый день. Это ~200+ сообщений/месяц только для анонсов обедов.**

### Паттерн: погодные решения в реальном времени
```
[08:45] Unyil: Batu talenan choppy.
[08:47] Unyil: Balik
[08:50] Abah Dani: Lady manta belok utara, gass utara.. 10 pax non swimmer on board 😅
[09:00] Abah Dani: Crystal gamat zonk...full utara gais
```
→ **Маршрут меняется в реальном времени через WhatsApp. Ресторан узнаёт об изменениях поздно или не узнаёт.**

### Паттерн: late cancellations (ежедневно)
```
[12:02] Gee DTB: Sanjaya 5 cancel lunch, tamunya ga mau lunch
[13:16] Gee DTB: Maaf, ralat# sesuai bookingan awal 7 pax
[16:45] Gee DTB: Pak grup LM cancel lunch
```
→ **Ресторан получает уведомления об отменах хаотично, иногда уже после того, как еда приготовлена.**

### Паттерн: pax discrepancy (регулярно)
```
[Kassy]: the red one is the invoice you haven't sent yet
[Fanny]: On this one i included only 5 pax as you said but my team counted 6
[Kassy]: Hi Fanny, one photographer [not counted in original pax]
```
→ **Каждый месяц — споры о количестве pax. Нет единой системы учёта, данные расходятся между бронированием, чатом и инвойсом.**

### Паттерн: ретрибуция — конец дня pressure
```
[16:04] Gede Retribusi: mohon untuk pembayarannya soalnya kami sudah closing
[16:04] Gee DTB: baik pak, tyang tf sekarang 🙏
[16:12] Gede Retribusi: ini sudah mau close @Gee DTB
```
