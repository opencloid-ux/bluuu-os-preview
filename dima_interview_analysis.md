## 2. Процесс до бронирования (боли)

- **Бот до брони = не работает.** Все 100% вручную: менеджеры отвечают на входящие
- Уточняют вручную: тур, дата, количество человек, возраст (чтобы определить private/shared)
- Если гость хочет private: ищут свободную лодку, присылают фото + описание (длина, capacity) — тоже вручную через «запуск бота»
- Получают email-уведомление с сайта → связываются → диалог
- **Оплата:** присылают ссылку на тур из Kommo → гость бронирует на сайте → данные возвращаются в лид (часть полей пустая, дозаполняют руками)

### Боль #1: нет автоматизации ДО бронирования
- Follow-up вручную
- Квалификация гостя вручную (private vs shared)
- Сбор данных о группе (возраст, количество) — вручную

## 3. Апсейл

- **Системного апсейла нет — скриптов нет никогда не было**
- Апсейл происходит только как «решение проблемы»:
  - Гость не заметил, что лодка на фото Premium, а забронировал Classic → предлагают апгрейд
  - Недобор на Premium → пересаживают с Classic Shared (Free Upgrade)
  - Во время чекина выясняется ребёнок <8 лет → апгрейд до Private
- **Flexible Booking** (300k/чел shared, 1M за лодку private): отправляется ботом после перевода в Booked. Конверсия очень низкая — «маркетингово не настроено»

### Боль #2: нет проактивного апсейла
- Нет скриптов, нет триггеров, нет автоматических предложений апгрейда
- Flexible Booking почти не продаётся — не продвигается

---

## 4. Отмены / Возвраты / No-Show

- Причины отмен **не фиксируются** — поле убрали из Kommo, «тормозило работу»
- No-show → идёт в cancellation без указания причины
- **Политика:**
  - No-show по вине гостя → **без возврата**, стриктли
  - Форс-мажор (война, вулкан, рейс отменён) → рескедж или полный возврат
  - «Человеческие» случаи (авария, болезнь) → по ситуации
  - Плохой отзыв → **рефанд за удаление** (Private ~20-25%, Shared ~50%)
- OTA cancellation: связываются через OTA (например Viator) с гостями +1 и +61 (США/Австралия)

### Боль #3: нет фиксации причин отмен
- Невозможно анализировать паттерны
- Рефанд-за-отзыв — дорогостоящая практика без системы, каждый раз вручную

## 5. WhatsApp Bot (текущий статус)

- **Бот работает ТОЛЬКО после бронирования** (post-booking):
  - Reminder накануне тура
  - Reminder утром в день тура
  - Web check-in напоминание
  - Запрос возрастов/национальностей (для OTA-букингов)
  - Flexible Booking предложение
  - После тура: ссылка на отзыв под конкретный booking source (через short URL)

- **Ломается при:**
  - Номера +1 (США) и +61 (Австралия) — Meta запрещает первым писать → ждут пока гость напишет сам
  - Ошибки Kommo → менеджеры дозаполняют вручную

### Боль #4: +1 и +61 номера — мёртвая зона
- Американские и австралийские гости выпадают из автоматизации
- Коммуникация через OTA-платформу для таких гостей — ручная работа

---

## 6. Отзывы (Review Management)

- Ответы: **не шаблонные** — Aline вручную через ChatGPT под каждую боль гостя
- **Скрипт работы с плохим отзывом:**
  1. Найти лид гостя
  2. Если нашли → предложить рефанд за удаление отзыва
  3. Если не нашли / отказал → написать в OTA с просьбой удалить (например «это не наш гость», «у нас нет public ferry»)
  4. Если OTA не удаляет → оставить профессиональный ответ с фактами
- Плохих отзывов мало: ~1/неделю, в high season 1-2/неделю
- Негатив во время тура → передают Geebran (guides) или Widia (lounge)

### Боль #5: нет CRM для review management
- Всё в разных OTA-кабинетах
- Нет единой очереди, нет трекинга статуса (нашли лид / написали в OTA / ответили)
