Модуль 4 · Sprint 1

+1 / +61 — Email Fallback.

Проблема

Meta запрещает первым писать американским (+1) и австралийским (+61) номерам. Это большой кусок рынка — США и Австралия топ-рынки Bluuu Tours.

1
Система определяет код страны при создании лида
2
Если +1 или +61 → канал переключается на EMAILEmail из OTA-букинга (всегда есть)
3
Pre-tour reminder → emailПараллельно: OTA-messaging через Viator/Klook API
4
Post-tour review request → emailБольше не теряем этих гостей автоматически
Почему важно
США и Австралия — одни из самых прибыльных рынков. Высокий средний чек, высокий LTV, хорошие отзывы. Терять их из-за ограничений Meta = терять деньги.
Для OTA-букингов
Отвечать через OTA Messaging API — не WhatsApp.
Viator, Klook, GYG имеют собственный messaging — используем его.
Сложность
Низкая — Sprint 1
Модуль 5 · Sprint 2

Review Pipeline.

Сейчас каждый отзыв — отдельная ручная работа в разных OTA-кабинетах. Будет единая очередь.

Как работает пайплайн
1
Агрегатор тянет отзывы из всех платформGoogle · Viator · GYG · Klook · Airbnb · TripAdvisor
2
Единая очередь по приоритету1★ → TOP · 2★ → High · 3★ → Medium
3
AI генерирует черновик ответаНа основе данных лида: лодка, гид, что произошло
4
Менеджер: читает → правит → отправляетНе пишет с нуля — только редактирует
5
Статус: New → Lead Found → Refund → Responded
Refund-for-review трекер
Фиксировать каждый случай: дата, сумма рефанда, источник, итог — удалил или нет.
Dashboard Кариму: сколько потрачено на «покупку удаления» за месяц → сигнал для решения.
Ты как Head of GR
Видишь не «ещё один отзыв», а очередь с приоритетами и контекстом. Знаешь: какой гид, какая лодка, что случилось, как ответить эффективно.
