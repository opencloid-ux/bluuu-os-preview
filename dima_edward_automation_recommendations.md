# Рекомендации по автоматизации: Дима + Эдвард
**На основе интервью 07.04.2026**

## 🔴 КРИТИЧНО — делать сейчас

### 1. Pre-booking AI-бот (до бронирования)
**Проблема:** Дима и Эдвард прямо говорят — до бронирования бот не делает ничего. Квалификация гостя, выбор private/shared, сбор данных о группе, ответы на вопросы — всё вручную.

**Решение:**
- Квалификационный флоу в боте: количество людей → возраст → дата → тип тура
- Авто-предложение Private если детей <8 лет или группа >8 чел.
- Авто-ответы на топ-5 вопросов Эдварда: цена, дата, вегетариан, трансфер включён?, погода
- База спецификаций лодок (год, размер, capacity, фото) — в боте, не вручную

### 2. OTA Auto-Input / Синхронизация
**Проблема:** Эдвард вводит OTA-букинги вручную каждый день → ошибки с датами (особенно Klook US format), риск двойного букинга.

**Решение:**
- API-интеграция с Viator, Klook, GYG, Airbnb → автоимпорт в Kommo/Odoo
- Unified availability: один источник истины для всех каналов
- Автоматическое закрытие/открытие слотов (уже в планах для Aline — теперь подтверждено ещё двумя)

### 3. Фиксация причин отмен
**Проблема:** Дима говорит — причины отмен не фиксируются, поле убрали из Kommo. Нет аналитики no-show, нет паттернов.

**Решение:**
- Обязательное поле "причина" в Respond.io при переводе в cancellation (быстрый select, не текст)
- Dashboard: % по причинам — no-show, weather, force-majeure, price dispute, age issue
- AI-разбор архивных чатов для восстановления исторических данных

## 🟠 ВЫСОКИЙ ПРИОРИТЕТ

### 4. Платёжная ссылка — один клик
**Проблема:** Раньше в Kommo был виджет → теперь нет → менеджер идёт на отдельный сайт, вручную вводит лид + цену.

**Решение:**
- Восстановить/заменить виджет в Respond.io: кнопка «Создать ссылку» → автозаполнение из лида → one-click

### 5. +1/+61 номера — обходной путь
**Проблема:** Дима говорит — американские и австралийские гости выпадают из всей автоматизации (Meta не разрешает первым писать).

**Решение:**
- Для +1/+61: авто-отправка через email (triggered из Respond.io при создании букинга)
- Или: OTA-сообщение вместо WhatsApp для этих гостей — автоматически

### 6. Проверка доступности — единый источник
**Проблема:** Эдвард проверяет сайт + calendar отдельно, потому что отмена на сайте не всегда освобождает слот.

**Решение:**
- Single source of truth: календарь = сайт = OTA — через API-интеграцию
- При cancellation в любом канале → авто-освобождение слота во всех

### 7. Погода в реальном времени в 6:30
**Проблема:** Эдвард ждёт когда кто-то из команды напишет погоду в 6:30. Нет автоматики.

**Решение:**
- Автоотчёт в GR group в 6:00: погода в Сарангане из API (wind speed, wave height) + GO/NO-GO статус
- (Уже пересекается с weather protocol модулем для David/Widia)

## 🟡 СРЕДНИЙ ПРИОРИТЕТ

### 8. Проактивный апсейл — скрипты и триггеры
**Проблема:** Дима говорит — апсейла как такового нет. Только как «решение проблемы».

**Решение:**
- Триггер: гость забронировал Classic Shared с группой >6 чел → авто-предложение Private
- Триггер: гость завтра едет на Shared, осталась лодка Premium с местами → авто-предложение Free Upgrade (с инфо о разнице)
- Flexible Booking: переформулировать как «страховка от изменений» → протестировать в боте до и после бронирования

### 9. Спецификации лодок — база знаний
**Проблема:** Эдвард не может быстро ответить на вопросы по лодкам — данных нет.

**Решение:**
- Таблица с характеристиками каждой лодки (год, длина, capacity, двигатель, фото)
- Доступна в боте как quick-reply + у менеджеров как база знаний

### 10. Review Management — единая очередь
**Проблема:** Дима и Aline работают с отзывами в разных OTA-кабинетах, нет трекинга.

**Решение:**
- Агрегатор отзывов (Google, Viator, GYG, Klook, Airbnb) → единая очередь с полем статуса
- Статусы: New / Found Lead / Offered Refund / Responded / Escalated to OTA / Resolved
- AI-черновик ответа с учётом боли гостя → Aline правит и отправляет

### 11. Local Agents — Virtual Account
**Проблема:** Эдвард ждёт ручного подтверждения от accounting для агентов после банковского перевода.

**Решение:**
- Virtual account (Xendit / Midtrans) для агентов → авто-подтверждение в системе

### 12. ChatGPT в процесс
**Проблема:** Эдвард использует ChatGPT лично для сложных гостей — не интегрировано.

**Решение:**
- Кнопка «AI Draft» в Respond.io для сложных/жалующихся гостей → AI-черновик → менеджер редактирует → отправляет
