🔗 Кросс-командные связи

Fleet не изолирован.
Он разрывает цепочку.

Fleet ↔ GR (Dima/Edward)
GR создаёт бронирование в Odoo — David не получает автоматически
Изменение / отмена брони → David узнаёт с задержкой
Pax в брони ≠ pax на борту — никто не сверяет до отплытия
→ Odoo webhook → авто-обновление манифеста при изменении
Fleet ↔ Guides (Geebran)
Гид назначен на тур → должен знать лодку, время, pax — из системы
SweetHeart4 чат: Geebran и David координируются вручную по каждому туру
Замена лодки → Geebran узнаёт из чата, не из дашборда
→ Единый дашборд: гид + лодка + pax + маршрут
Fleet ↔ Finance (Karim)
Топливо, ТО, операционные расходы — в свободном тексте WA
Karim добавлен в Boat Security — значит финансовый контроль важен
Нет экспорта в бухгалтерию. Каждый расчёт = ручная работа
→ Fuel + maintenance log → авто-экспорт в Odoo Finance
Главный системный разрыв
GR создаёт данные в Odoo → Fleet никогда их не получает автоматически. Каждое утро David вручную собирает информацию из нескольких источников. Auto-manifest = связующее звено всей системы.
