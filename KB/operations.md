## 1. Weather GO/NO-GO протокол

### Источники данных
- Windy
- BMKG (Индонезийский метеосервис)
- Оценка капитана / физические замеры на месте

### Пороговые значения NOGO
| Параметр | Порог стоп |
|---|---|
| Ветер | >20 узлов |
| Волна | >1.5 м |

### "Rule of Two No's"
Тур отменяется автоматически, если **2 из 3** источников (Windy / BMKG / капитан) показывают небезопасные условия. Решение финальное, без дополнительного обсуждения.

### Тайминг принятия решения
| Время | Действие |
|---|---|
| 20:00 (накануне) | Предварительная проверка прогноза |
| 05:00–05:30 (утро тура) | Актуальные данные из всех источников |
| 05:45 | Финальное решение — Head of Boats |
| 06:00–06:30 | Диспетчеризация / уведомление команды |

### Сезонные различия
- **Сухой сезон (апр–окт):** стандартные пороги
- **Влажный сезон (ноя–мар):** пороги жёстче, выше вероятность NOGO

### Коммуникация при отмене
- Гости получают WhatsApp-уведомление в течение 15 мин после решения NOGO
- Если NOGO принято вечером — сообщить вечером, не откладывать на утро
- Всегда предлагать два варианта: **бесплатный перенос** или **полный возврат**
- Если гость уже едет в порт — обязательно звонить
- Решение фиксируется в Weather Decision Log

## 2. Лаунж и встреча гостей (от Widia, интервью 02.04.2026)

### Роль Widia
Widia — глава лаунжа, отвечает за гостевой опыт с момента прихода гостя до отправления на лодке и по возвращении.

### Утренняя подготовка лаунжа
- Проверка состояния лаунжа: чистота, порядок, готовность к встрече гостей
- Координация с операционным отделом: расписание лодок, аменити, включения
- Проверка дополнительных заказов: напитки, сырные тарелки, завтрак, торты, цветы для пар

### Поток гостей по времени (morning window)
| Время | Тип тура |
|---|---|
| 07:30 | Shared Standard — прибытие |
| 08:00–08:15 | Shared Premium — прибытие |
| 08:30–09:30 | Private и Private Premium — прибытие |

### Объём гостей
- **Низкий / средний сезон:** 30–60 гостей в день
- **Пиковый сезон:** 60–120+ гостей в день

### Процесс check-in (по словам Widia)
1. Приветствие гостей
2. Проверка ваучеров бронирования
3. Подтверждение типа тура и доплаты (balance payment)
4. Объяснение включений: полотенца, снаряжение для снорклинга
5. Бриф по маршруту (карта на стене)
6. Идентификация: браслеты или ланьярды для посадки на лодку

**Текущая проблема:** чекин частично ручной — распечатка из CRM + ручные записи. Нужна автоматизация.

### Встреча по возвращении (afternoon)
- Холодные полотенца (осибори)
- Освежающие напитки: холодный чай, infused water
- Свежие полотенца
- Живое общение: Widia спрашивает гостей о впечатлениях

### Стиль работы Widia
> *"I prefer to stay present on the floor to observe, approach guests when needed, and support the team — because small things like greeting guests and responding quickly impact overall experience."*

## 3. Типичные жалобы и как с ними работают (от Widia)

### Топ жалоб на лаунже

| Жалоба | Описание | Решение |
|---|---|---|
| Лодка не соответствует ожиданиям | Гость не понял что забронировал / как выглядит лодка | Улучшение pre-trip информации, чёткий бриф при чекине |
| Путаница shared vs private | Гость думал, что тур приватный | Подтверждение типа тура при чекине |
| Погодные отмены / задержки | Почему не предупредили заранее? | Ранняя коммуникация, скрипт: "ждём решения harbour master, держим вас в курсе" |

### Как Widia обрабатывает жалобы
1. Сохранять спокойствие, слушать — гость должен почувствовать, что его слышат
2. Признать проблему, проявить эмпатию
3. Объяснить ситуацию честно
4. Предложить решение: перенос / корректировка плана / комфорт в лаунже
5. Коммуникация важнее результата — как говоришь, меняет восприятие

> *"Even if we can't fully solve the issue, how we communicate can change how guests feel."*

### Работа с повторяющимися жалобами
- Widia фиксирует повторяющиеся проблемы
- Обсуждает на еженедельных / двухнедельных митингах с операционным отделом
- Ищет превентивные решения

**Пример:** задержки в пиковый сезон из-за перегруженности гавани → теперь предупреждают гостей о возможных задержках заранее.

## 7. Процесс бронирования

**Полный флоу:**
```
Запрос (сайт / WhatsApp / OTA)
  ↓
Первичный ответ <15 мин (рабочие часы)
  ↓
Согласование даты, типа тура, наличия
  ↓
Депозит 50% (карта / PayPal / наличные)
  ↓
Подтверждение + ссылка на онлайн чекин
  ↓
Онлайн чекин: имена, возраст, гражданство
  ↓
Напоминание T-24ч (авто, через бота)
  ↓
Водитель пишет до 21:00 накануне
  ↓
День тура: встреча в лаунже → чекин → лодка
  ↓
Возвращение: осибори, напитки, общение
  ↓
Follow-up + запрос отзыва (авто, через бота)
```

**Роли:**
- Widia / GR лаунж — физический чекин, встреча, проводы
- Дима / GR менеджеры — коммуникация с гостями
- Бот (Respond.io) — авто-напоминания, FAQ, follow-up

## 8. Guest Relations SOP

### Типы обращений
| Тип | Workflow | Эскалация |
|---|---|---|
| Запрос цены / наличия | Авто-бот → GR менеджер | — |
| Жалоба на сервис / лодку | GR → Дима / Карим | Да |
| Погодная отмена | Скрипт + перенос / рефанд | Нет |
| Фото / GoPro не получены | GR → менеджер / фотограф | — |
| Проблема с трансфером | Менеджер → водитель | Да |
| Агрессивный гость | Сразу Дима / Карим | Да |

### Политика работы с негативными отзывами
- Стандарт: **50% рефанд за удаление отзыва** на OTA
- Все сложные кейсы — только через опытного менеджера

## 9. Политика отмен и возвратов

| Ситуация | Возврат |
|---|---|
| Отмена за 24ч+ | Полный возврат |
| Отмена менее 24ч | Нет (депозит не возвращается) |
| NOGO по погоде | Полный возврат или бесплатный перенос |
| Болезнь / форс-мажор | Частичный возврат по решению менеджера |
| Перенос даты | Бесплатно если за 24ч+ |

Все возвраты — за вычетом банковских / OTA комиссий.
