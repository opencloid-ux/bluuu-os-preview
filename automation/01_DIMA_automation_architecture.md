## Архитектура автоматизации

### Модуль 1: Pre-Booking AI Bot
**Статус сейчас:** ноль — весь пре-букинг вручную  
**Что делаем:**

```
Гость пишет в WhatsApp/Instagram/сайт
         ↓
[AI Bot] — квалификационный флоу:
  → сколько человек?
  → дата?
  → есть дети до 8 лет? (если да → только Private, объяснить почему)
  → Private или Shared? (если не знают → презентация разницы с фото)
         ↓
[AI Bot] — проверяет availability в реальном времени (Odoo API)
         ↓
[AI Bot] — отправляет персонализированную рекомендацию:
  → лодка + фото + capacity + цена
  → booking link
         ↓
[AI Bot] — follow-up через 24/48/72 ч если не оплатил
         ↓
При оплате → автоматический перевод в post-booking флоу
```

**Топ-10 автоответов (на основе 1.1M строк CRM):**
1. Трансфер/пикап → где meeting point, время
2. Маршрут и острова → что включено
3. Наличие на дату → real-time проверка
4. Цена → показать все тиры
5. Private vs Shared → презентация разницы
6. Снорклинг/манты → что ожидать
7. Питание → вегетариан, аллергены
8. Дети/возраст → политика
9. Оплата → как оплатить
10. Погода → стандартный ответ + ссылка на политику отмен

**ROI:** 80% входящих — эти 10 тем. Автоматизация снимет ~60% нагрузки с команды GR.

### Модуль 2: Фиксация причин отмен

**Проблема:** поле убрали из Kommo потому что «тормозило работу».  
**Решение:** quick-select в Respond.io — 1 нажатие, не текст:

```
При переводе лида в статус "Cancelled":
  → обязательное поле (нельзя пропустить):
    [ ] Болезнь гостя
    [ ] Страх плохой погоды
    [ ] Нашли дешевле
    [ ] Изменение планов
    [ ] Уменьшение группы
    [ ] No-show
    [ ] Отмена по вине компании
    [ ] Другое (текст)
  → автоматически записывается в Odoo
```

**Dashboard для Димы:**
- % отмен по причинам (week/month)
- Топ дни с no-show → выявить паттерны (определённые рынки? определённые OTA?)
- Динамика cancel rate по каналам

### Модуль 3: Система апсейла

**Сейчас:** апсейл случается только как «решение проблемы» (ребёнок <8 лет, wrong boat).

**Триггеры автоматического апсейла:**

| Триггер | Действие |
|---------|----------|
| Группа 6+ человек на Shared | Бот: «For your group size, Private is actually more cost-effective. Here's why...» |
| Ребёнок <8 лет в группе | Немедленный апгрейд до Private — обязательно, объяснить безопасность |
| Дата тура через 5+ дней | Предложение Flexible Booking как «страховки от изменений» |
| Premium Shared < 50% загрузки за 48ч до тура | Автоопция: upgrade с Classic за Х IDR доплаты |
| Гость из USA/Australia/EU (высокий LTV) | Приоритетное предложение Private первым |

**Flexible Booking ребрендинг:**
- Текущий: «Flexible Booking — 300K/чел» → конверсия ~0%
- Новый фрейм: «✈️ Travel Insurance for your tour — reschedule or cancel up to 24h before, zero risk»
- Выводить в боте ПОСЛЕ confirmation, не после payment

### Модуль 4: +1/+61 Email Fallback

**Проблема:** Meta запрещает первым писать американским (+1) и австралийским (+61) номерам.

**Решение:**
```
При создании лида:
  → система определяет код страны номера
  → если +1 или +61:
    → канал коммуникации переключается на EMAIL
    → pre-tour reminder → отправляется на email (из OTA букинга)
    → post-tour review request → email
  → параллельно: OTA-messaging через API (Viator/Klook)
```

**Дополнительно:** для OTA-букингов — отвечать через OTA messaging API, не WhatsApp.

### Модуль 5: Review Management Pipeline

**Сейчас:** каждый плохой отзыв — отдельная ручная работа в разных OTA-кабинетах.

```
[Review Aggregator] — тянет отзывы из:
  Google / Viator / GYG / Klook / Airbnb / TripAdvisor
         ↓
[Dashboard] — единая очередь, ранжированная по приоритету:
  1⭐ отзывы → TOP priority
  2⭐ → High
  3⭐ → Medium
         ↓
[AI Draft] — черновик ответа под конкретную боль гостя
  (на основе данных из лида: какая лодка, какой гид, что случилось)
         ↓
Aline: review + edit + submit
         ↓
[Status tracker]:
  [ ] New → [ ] Lead Found → [ ] Refund Offered → [ ] Responded → [ ] Resolved
```

**Refund-for-review tracking:**
- Фиксировать каждый случай: дата, сумма рефанда, источник, итог (удалил / не удалил)
- Dashboard: сколько денег потрачено на «покупку удаления» за месяц → сигнал Кариму
